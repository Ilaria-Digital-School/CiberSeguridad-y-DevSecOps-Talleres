# Taller de variantes: creación de tus propios ejemplos de amenazas

RECORDATORIO TEÓRICO Antes de comenzar el taller, repasa los conceptos fundamentales que has trabajado en los módulos anteriores. La seguridad informática descansa sobre cuatro pilares conceptuales: seguridad (estado global que se desea alcanzar), amenaza (acción o evento potencial que puede causar daño), vulnerabilidad (debilidad explotable en un sistema) y riesgo (probabilidad de que una amenaza se materialice aprovechando una vulnerabilidad). Estos conceptos se articulan con el modelo CIA: Confidencialidad (solo los usuarios autorizados acceden a la información), Integridad (la información no es alterada de forma no autorizada) y Disponibilidad (los servicios e información son accesibles cuando se necesitan). Un error frecuente entre los principiantes es confundir amenaza y vulnerabilidad. La amenaza es lo que podría ocurrir (un atacante que intenta robar contraseñas); la vulnerabilidad es la condición que lo hace posible (una base de datos sin cifrar). El riesgo es la combinación de ambos. Mini-ejemplo: una aplicación web que almacena contraseñas en texto plano (vulnerabilidad) es susceptible de sufrir una filtración de credenciales por parte de un atacante externo (amenaza), lo que compromete la Confidencialidad del pilar CIA. Ten presente este triángulo conceptual amenaza-vulnerabilidad-riesgo y el modelo CIA durante todo el taller, ya que serán la brújula de tu análisis. --- ENUNCIADO DEL TALLER Este taller tiene como objetivo consolidar tu comprensión del vocabulario esencial de ciberseguridad y del modelo CIA, forzándote a salir del rol pasivo de receptor de información para adoptar el rol activo de creador de ejemplos.
