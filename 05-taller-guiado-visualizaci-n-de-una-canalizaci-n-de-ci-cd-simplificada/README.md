# Taller guiado: visualización de una canalización de CI/CD simplificada

Recordatorio teórico (≈185 palabras): Una canalización de Integración Continua / Entrega Continua (CI/CD) es la secuencia automatizada que traslada el código desde el repositorio hasta un entorno listo para producción. Incluye al menos cuatro fases: compilación, pruebas (unitarias y de seguridad), empaquetado y despliegue. La mentalidad clave es detectar defectos lo antes posible; cuanto más a la izquierda se descubra un fallo, menor será el coste de corregirlo. Los principiantes suelen cometer tres errores: confundir pasos de integración con pasos de entrega, olvidar almacenar los artefactos generados y mezclar credenciales dentro del código fuente. Recuerda el mini-ejemplo visto en clase: un commit en GitHub activa un flujo en GitHub Actions que compila una app Node, ejecuta ESLint, SAST con Semgrep y, si todo es verde, crea una imagen Docker etiquetada con el número de commit. Al final, el pipeline publica el artefacto en un registro privado y lo despliega a un servidor de pruebas. Observa cómo los chequeos de seguridad están insertados justo después de la compilación: esa es la esencia de DevSecOps. Enunciado general: Tu misión consiste en dibujar y comentar una canalización de CI/CD simplificada para un proyecto web de tu elección (real o ficticio).
