# Taller: instalación y verificación de la máquina virtual del laboratorio

**RECORDATORIO TEÓRICO** Una máquina virtual (VM) es un entorno de computación aislado que emula un sistema operativo completo sobre una máquina física real. Gracias a un hipervisor (como VirtualBox o VMware), es posible ejecutar varios sistemas operativos de forma simultánea y completamente separada. En ciberseguridad, este aislamiento es fundamental: permite realizar pruebas, ataques simulados y experimentos sin comprometer el sistema operativo principal ni la red de producción. Los dos conceptos clave que debes interiorizar son: el **aislamiento** (la VM no puede afectar al sistema anfitrión si está bien configurada) y la **reproducibilidad** (puedes restaurar una instantánea —snapshot— en segundos si algo sale mal). Un error frecuente en principiantes es trabajar directamente en su máquina personal, exponiendo sus datos reales a herramientas ofensivas. Otro error habitual es no verificar la firma o el hash del archivo ISO descargado, lo que puede comprometer toda la cadena de trabajo desde el inicio. Mini-ejemplo: si descargas Kali Linux y no verificas el hash SHA256, podrías instalar una imagen alterada. La verificación es tan simple como ejecutar `sha256sum kali-linux.iso` y comparar el resultado con el valor publicado oficialmente en el sitio de Kali.
