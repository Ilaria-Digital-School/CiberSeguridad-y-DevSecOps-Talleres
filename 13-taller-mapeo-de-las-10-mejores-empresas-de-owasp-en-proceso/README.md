# Taller: mapeo de las 10 mejores empresas de OWASP en proceso

**RECORDATORIO TEÓRICO** El OWASP Top 10 es una lista de referencia mundial que identifica las diez categorías de vulnerabilidades más críticas en aplicaciones web. Publicada y mantenida por la comunidad OWASP (Open Web Application Security Project), esta lista no es simplemente un inventario de errores: es una brújula para priorizar esfuerzos de seguridad en proyectos reales. Las categorías más recientes incluyen riesgos como el control de acceso roto (A01), fallas criptográficas (A02), inyecciones (A03), diseño inseguro (A04), mala configuración de seguridad (A05), componentes vulnerables (A06), fallas de identificación y autenticación (A07), fallas de integridad de software y datos (A08), registros y monitoreo insuficientes (A09) y falsificación de solicitudes del lado del servidor o SSRF (A10). El modelo mental clave es el siguiente: cada riesgo OWASP no existe en abstracto, sino que se materializa en pasos concretos de un proceso de desarrollo o pipeline CI/CD. Por ejemplo, una credencial hardcodeada en el código fuente corresponde a A02 y puede detectarse en la fase de SAST. Un error frecuente de los principiantes es memorizar los nombres de los riesgos sin vincularlos a situaciones concretas de desarrollo o despliegue. Mini-ejemplo: si un pipeline de CI/CD no valida las dependencias de terceros antes de construir la imagen Docker, el riesgo A06 (componentes vulnerables) está latente y explotable en producción.
