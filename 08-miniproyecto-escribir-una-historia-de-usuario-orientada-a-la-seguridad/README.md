# Miniproyecto : escribir una historia de usuario orientada a la seguridad

## Descripción del proyecto En este miniproyecto, el estudiante deberá redactar un conjunto de historias de usuario orientadas a la seguridad para una aplicación web de comercio electrónico ficticia. Este ejercicio integra los conocimientos adquiridos sobre DevSecOps, las etapas del pipeline CI/CD, la tríada CIA (Confidencialidad, Integridad, Disponibilidad) y los principios de seguridad aplicados al desarrollo de software. Una historia de usuario orientada a la seguridad (también llamada 'security user story' o 'abuser story' en algunos contextos) no solo describe lo que un usuario legítimo desea hacer, sino que también anticipa los riesgos de seguridad asociados y define los criterios de aceptación que garantizan que la funcionalidad se implementa de forma segura. ## Contexto del proyecto Eres miembro del equipo DevSecOps de una startup de comercio electrónico llamada **SecureShop**. La empresa está construyendo su plataforma de ventas en línea y necesita que el equipo de desarrollo integre la seguridad desde las primeras etapas del ciclo de vida del software (principio 'Shift Left'). Tu misión es redactar un conjunto de historias de usuario orientadas a la seguridad que servirán como base para el backlog del producto, asegurando que cada funcionalidad contemple explícitamente los requisitos de seguridad. ## Competencias a validar 1. Comprender e integrar los principios DevSecOps en el ciclo de vida del desarrollo de software. 2. Aplicar la tríada CIA (Confidencialidad, Integridad, Disponibilidad) en la redacción de requisitos funcionales. 3. Identificar amenazas potenciales asociadas a funcionalidades de una aplicación web. 4. Traducir requisitos de seguridad en historias de usuario concretas...
