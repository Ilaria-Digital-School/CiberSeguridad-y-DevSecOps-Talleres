# Taller: visualización de una canalización de CI/CD simplificada

**RECORDATORIO TEÓRICO** Antes de comenzar el taller, repasemos los conceptos esenciales. DevOps es una cultura y un conjunto de prácticas que acercan los equipos de desarrollo (Dev) y operaciones (Ops) para entregar software de manera más rápida y fiable. DevSecOps integra la seguridad (Sec) en ese proceso desde el primer momento, siguiendo el principio de 'shift left': detectar y corregir problemas de seguridad lo antes posible en el ciclo de vida del software, donde son más baratos de solucionar. Una canalización CI/CD (Integración Continua / Entrega Continua) es el conjunto automatizado de etapas que transforma el código fuente escrito por un desarrollador en una aplicación desplegada en producción. Las etapas típicas son: escritura del código → commit en repositorio → compilación/build → pruebas automáticas → análisis de seguridad → empaquetado → despliegue. En DevSecOps, se insertan controles de seguridad específicos en estas etapas: SAST (análisis estático del código), SCA (análisis de dependencias) y DAST (pruebas dinámicas sobre la aplicación en ejecución). Error frecuente: confundir CI (integración continua, que valida el código automáticamente) con CD (entrega/despliegue continuo, que lleva ese código validado a producción). Son etapas consecutivas, no sinónimas. Mini-ejemplo: un desarrollador sube un cambio de código → el sistema ejecuta automáticamente los tests unitarios → si pasan, ejecuta un SAST → si no hay vulnerabilidades críticas, empaqueta la aplicación → la despliega en un entorno de pruebas.
