# Taller de variantes: creación de tus propios ejemplos de amenazas

Recordatorio teórico (≈190 palabras) Antes de crear variantes propias conviene afianzar el modelo mental visto en las sesiones anteriores. Una AMENAZA describe la intención o circunstancia que puede explotar una VULNERABILIDAD e impactar sobre un activo comprometido con los pilares CIA. El RIESGO es la combinación de probabilidad y impacto cuando la amenaza se materializa. Error frecuente: confundir vulnerabilidad con amenaza ("SQL Injection" es la vulnerabilidad; "atacante extrae datos" es la amenaza). Otro desliz habitual es imaginar amenazas demasiado genéricas ("hacker externo"). El truco es pensar en "agente + acción + objetivo + consecuencia". Mini-ejemplo: "Empleado descontento (agente) descarga un script de borrado (acción) sobre el servidor de backups (objetivo) y provoca pérdida de integridad y disponibilidad (consecuencia)". Mientras diseñes tus ejemplos pregúntate siempre: ¿qué pilar de la CIA sufre?, ¿cómo cambia la probabilidad si añado o quito controles?, ¿qué vector de ataque concreto se usa? Enunciado del taller Debes elaborar TU PROPIO CATÁLOGO DE AMENAZAS VARIANTES para una aplicación web ficticia de comercio electrónico (frontend React + API REST + base de datos MySQL). El objetivo es que apliques el vocabulario y aprendas a descomponer una situación real en piezas accionables para un posterior Pentest. Entregarás entre 8 y 12 amenazas diferentes agrupadas en al menos 3 familias (p. ej. autenticación, inyección, CSRF). Cada amenaza irá acompañada de: (1) descripción breve en formato "agente + acción + objetivo + consecuencia", (2) pilar de la CIA afectado, (3) vulnerabilidad presumible, (4) nivel de riesgo estimado (bajo/medio/alto) con justificación en una...
