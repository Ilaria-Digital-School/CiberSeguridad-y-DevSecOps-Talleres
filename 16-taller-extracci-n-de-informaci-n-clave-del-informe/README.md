# Taller: extracción de información clave del informe

**Recordatorio teórico** Un informe de vulnerabilidad es un documento estructurado que comunica los hallazgos de una evaluación de seguridad a las partes interesadas técnicas y no técnicas. Su correcta lectura e interpretación es una competencia fundamental para cualquier especialista en ciberseguridad. Los elementos clave de un informe incluyen: el **resumen ejecutivo** (panorama general para no técnicos), la **descripción técnica de la vulnerabilidad** (cómo se detectó y por qué existe), el **nivel de severidad** (usualmente basado en CVSS: Critical, High, Medium, Low, Informational), el **impacto potencial** (sobre confidencialidad, integridad y disponibilidad, es decir, la triada CIA), el **vector de ataque** (cómo un atacante podría explotarla), la **evidencia** (capturas, logs, payloads) y las **recomendaciones de remediación**. Un error frecuente entre principiantes es confundir la severidad con la prioridad de corrección: una vulnerabilidad de severidad media en un sistema crítico puede requerir atención inmediata, mientras que una crítica en un sistema aislado puede esperar. Otro error habitual es ignorar el contexto de negocio al interpretar el impacto. El modelo mental correcto es leer el informe como un mapa de riesgos, no como una lista de tareas técnicas. Mini-ejemplo: si un informe indica "SQL Injection en el endpoint /api/login, CVSS 9.8, impacto sobre confidencialidad: alto", el analista debe comprender que cualquier usuario no autenticado puede potencialmente extraer toda la base de datos, y que la remediación debe ser inmediata. --- **Enunciado del taller** En el capítulo anterior trabajaste con la lectura de un informe de vulnerabilidad ficticio, donde te familiarizaste con su estructura y...
