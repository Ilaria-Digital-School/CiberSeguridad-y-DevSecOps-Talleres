# Taller guiado: asocia cada término con un caso concreto

RECORDATORIO TEÓRICO Antes de comenzar, repasa los cuatro conceptos fundamentales que trabajaste en el capítulo anterior: — Seguridad: conjunto de medidas técnicas y organizativas destinadas a proteger sistemas, datos y usuarios frente a acciones no deseadas. — Amenaza: cualquier evento o actor (interno o externo) que podría causar daño a un sistema. Ejemplo: un atacante que intenta acceder a una base de datos mediante credenciales robadas. — Vulnerabilidad: debilidad técnica o humana que puede ser explotada por una amenaza. Ejemplo: un formulario web que no filtra las entradas del usuario y permite inyecciones SQL. — Riesgo: la combinación de la probabilidad de que una amenaza explote una vulnerabilidad y el impacto potencial resultante. El riesgo no desaparece; se gestiona. Modelo mental clave: Riesgo = Amenaza × Vulnerabilidad × Impacto. Si alguno de estos factores es cero, el riesgo se anula o se reduce drásticamente. Error frecuente: confundir amenaza con vulnerabilidad. La amenaza es el agente o evento externo (el atacante, el malware); la vulnerabilidad es la brecha interna (la contraseña débil, el software desactualizado). El riesgo es la consecuencia de que ambos coincidan. Mini-ejemplo: Una empresa publica una API REST sin autenticación (vulnerabilidad). Un bot automatizado la descubre y extrae datos de clientes (amenaza). El riesgo es la exposición masiva de datos personales con consecuencias legales y reputacionales. --- ENUNCIADO DEL TALLER Este taller tiene como objetivo consolidar tu comprensión de los cuatro términos esenciales de ciberseguridad — seguridad, amenaza, vulnerabilidad y riesgo — aplicándolos a situaciones concretas del mundo...
